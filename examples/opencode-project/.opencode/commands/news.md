使用 HLTV MCP 查询新闻集合。

默认假设当前 OpenCode 中的 MCP 名称是 `hltv_local`，所以工具名前缀是 `hltv_local_`。
如果你的 MCP 名称不是 `hltv_local`，请先把下面工具名替换成你的真实前缀再执行。

执行规则：

1. 默认新闻请求不要传 `tag`；不要把 `news`、`latest`、`today`、`新闻`、`今日新闻`、`最新新闻` 之类泛化词映射成 `tag`。
2. 当前归档工具是 `hltv_local_hltv_news_digest`；默认参数只传：
   - `limit`: `25`
3. 如果用户回复“继续”，优先沿用上一轮查询条件，并优先使用上一轮工具返回的 `next_offset` 或 `next_page` 继续取下一批 25 条。
4. 输出时保留：
   - 中文标题（由命令执行时的助手基于英文原标题生成简洁中文标题；工具原始结果仍以英文标题为准）
   - 英文原标题
   - 发布时间
   - 主题/标签（可选，有则保留，无则省略）
   - 更新时间（统一按 `Asia/Shanghai` 展示）
5. 不要输出数据源字段。
6. 如果没有数据，明确说明是“暂无匹配新闻”。
7. 如果工具返回错误，原样说明错误码和事实上游信息，不要脑补。

用户输入：
$ARGUMENTS
