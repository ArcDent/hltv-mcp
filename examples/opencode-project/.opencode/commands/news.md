使用 HLTV MCP 查询实时新闻。

默认假设当前 OpenCode 中的 MCP 名称是 `hltv_local`，所以工具名前缀是 `hltv_local_`。
如果你的 MCP 名称不是 `hltv_local`，请先把下面工具名替换成你的真实前缀再执行。

执行规则：

1. 默认新闻、最新新闻、今日新闻、实时新闻请求固定调用 `hltv_local_hltv_realtime_news`。
2. 不要传 `tag`；`hltv_realtime_news` 没有 `tag` 入参，分类/主题由爬虫从 HLTV 实时新闻流中提取。
3. 默认参数只传：
   - `limit`: `25`
4. 如果用户回复“继续”，优先沿用上一轮查询条件，并优先使用上一轮工具返回的 `next_offset` 或 `next_page` 继续取下一批 25 条。
5. 只有用户明确要求“归档新闻”“某年某月新闻”或指定历史月份时，才改用 `hltv_local_hltv_news_digest`；该归档工具默认也不要传 `tag`，除非用户明确给出具体队伍、赛事或主题过滤词。
6. 输出时保留：
   - 中文标题（由命令执行时的助手基于英文原标题生成简洁中文标题；工具原始结果仍以英文标题为准）
   - 英文原标题
   - 相对发布时间
   - 分类/主题（可选，有则保留，无则省略）
   - 评论数（可选，有则保留，无则省略）
   - 更新时间（统一按 `Asia/Shanghai` 展示）
7. 不要输出数据源字段。
8. 如果没有数据，明确说明是“暂无匹配新闻”。
9. 如果工具返回错误，原样说明错误码和事实上游信息，不要脑补。

用户输入：
$ARGUMENTS
